### 2.2. Requerimientos de Atributos de Calidad

**Disponibilidad (RNF01):**

La disponibilidad del sistema es crucial para asegurar que los usuarios puedan acceder a la plataforma en cualquier momento, especialmente durante los periodos de alta demanda académica.

| Campo           | Descripción                                                                                          |
|-----------------|------------------------------------------------------------------------------------------------------|
| **Escenario**   | El sistema debe estar disponible el 99.5% del tiempo, especialmente durante los periodos de actividad académica. |
| **Estímulo**    | Un usuario intenta acceder al sistema durante un periodo de alta demanda.                            |
| **Fuente**      | Necesidad del usuario de utilizar la plataforma en un momento crucial.                               |
| **Artefacto**   | Infraestructura de servidores.                                                                       |
| **Respuesta**   | Implementar redundancia y balanceo de carga para asegurar la disponibilidad del sistema.             |
| **Medida**      | Porcentaje de tiempo en que el sistema está disponible.                                              |

**Rendimiento (RNF02):**

El rendimiento del sistema asegura que la plataforma puede manejar un gran número de usuarios concurrentes sin que su funcionalidad se vea comprometida.

| Campo           | Descripción                                                                                          |
|-----------------|------------------------------------------------------------------------------------------------------|
| **Escenario**   | El sistema debe ser capaz de manejar hasta 5000 usuarios concurrentes sin degradación significativa del rendimiento. |
| **Estímulo**    | Un gran número de usuarios intenta acceder y usar la plataforma simultáneamente.                     |
| **Fuente**      | Necesidad del usuario de utilizar la plataforma de manera eficiente.                                 |
| **Artefacto**   | Base de datos y servidor de aplicaciones.                                                            |
| **Respuesta**   | Optimización de consultas y escalabilidad horizontal para manejar la carga.                          |
| **Medida**      | Tiempo de respuesta del sistema y capacidad de manejo de usuarios concurrentes.                      |

**Seguridad (RNF03):**

La seguridad del sistema es fundamental para proteger la información sensible de los usuarios y garantizar la integridad de la plataforma.

| Campo           | Descripción                                                                                          |
|-----------------|------------------------------------------------------------------------------------------------------|
| **Escenario**   | El sistema debe implementar autenticación de dos factores y cifrado de datos en reposo y en tránsito para proteger la información del usuario. |
| **Estímulo**    | Un usuario intenta iniciar sesión en la plataforma.                                                  |
| **Fuente**      | Necesidad del usuario de acceder al sistema de forma segura.                                         |
| **Artefacto**   | Autenticación y almacenamiento de datos.                                                             |
| **Respuesta**   | Implementar cifrado AES para datos en reposo, SSL/TLS para datos en tránsito, y autenticación multifactor. |
| **Medida**      | Número de brechas de seguridad y efectividad de la autenticación.                                    |

**Usabilidad (RNF04):**

La usabilidad del sistema asegura que la plataforma sea intuitiva y fácil de usar, proporcionando una experiencia positiva a los usuarios.

| Campo           | Descripción                                                                                          |
|-----------------|------------------------------------------------------------------------------------------------------|
| **Escenario**   | La plataforma debe ser intuitiva y fácil de usar para que los estudiantes y otros usuarios puedan navegar y utilizar las funciones sin dificultad. |
| **Estímulo**    | Un nuevo usuario intenta usar la plataforma por primera vez.                                         |
| **Fuente**      | Necesidad del usuario de comprender rápidamente cómo utilizar la plataforma.                         |
| **Artefacto**   | Interfaz de usuario.                                                                                 |
| **Respuesta**   | Realización de pruebas de usabilidad y diseño centrado en el usuario para garantizar una experiencia de usuario positiva. |
| **Medida**      | Tiempo que tarda un usuario en completar una tarea y satisfacción del usuario con la interfaz.       |

**Escalabilidad (RNF05):**

La escalabilidad del sistema es importante para manejar el crecimiento futuro y la expansión de la base de usuarios.

| Campo           | Descripción                                                                                          |
|-----------------|------------------------------------------------------------------------------------------------------|
| **Escenario**   | El sistema debe poder expandirse para soportar un aumento en la cantidad de usuarios y datos sin afectar el rendimiento. |
| **Estímulo**    | Aumento significativo en el número de usuarios registrados y activos.                                |
| **Fuente**      | Necesidad de la plataforma de manejar un crecimiento exponencial.                                     |
| **Artefacto**   | Infraestructura de servidores y base de datos.                                                       |
| **Respuesta**   | Implementar soluciones de escalabilidad horizontal y vertical, así como balanceo de carga.           |
| **Medida**      | Capacidad del sistema para manejar usuarios adicionales y volúmenes de datos sin pérdida de rendimiento. |

**Compatibilidad (RNF06):**

La compatibilidad del sistema con diferentes dispositivos y navegadores es esencial para asegurar el acceso universal.

| Campo           | Descripción                                                                                          |
|-----------------|------------------------------------------------------------------------------------------------------|
| **Escenario**   | El sistema debe ser compatible con los principales navegadores web y dispositivos móviles.            |
| **Estímulo**    | Usuarios intentan acceder a la plataforma desde diferentes dispositivos y navegadores.               |
| **Fuente**      | Necesidad de los usuarios de acceder a la plataforma desde múltiples entornos.                       |
| **Artefacto**   | Interfaz de usuario y backend de la plataforma.                                                      |
| **Respuesta**   | Realizar pruebas de compatibilidad y ajustar el diseño responsivo para asegurar un funcionamiento óptimo. |
| **Medida**      | Número de problemas de compatibilidad reportados y nivel de satisfacción del usuario con el acceso desde diferentes dispositivos. |

### 2.3. Restricciones

**Tecnologías Propias:**

| Campo           | Descripción                                                                                          |
|-----------------|------------------------------------------------------------------------------------------------------|
| **Descripción** | Nosotros desarrollamos todas las tecnologías internamente, incluyendo el marco de trabajo y las políticas de seguridad. |
| **Sustento**    | Garantizar un control total sobre la calidad y la seguridad del software.                            |

**Base de Datos Relacional:**

| Campo           | Descripción                                                                                          |
|-----------------|------------------------------------------------------------------------------------------------------|
| **Descripción** | Utilizaremos una base de datos relacional para almacenar datos críticos del usuario y los contenidos educativos. |
| **Sustento**    | Mejor manejo de relaciones complejas y consultas eficientes.                                         |

**Servidores en la Nube:**

| Campo           | Descripción                                                                                          |
|-----------------|------------------------------------------------------------------------------------------------------|
| **Descripción** | Todos los servidores y recursos de la plataforma estarán desplegados en la nube para garantizar escalabilidad y accesibilidad. |
| **Sustento**    | Facilitar la expansión y el acceso global sin necesidad de infraestructura física.                   |

**Política de Seguridad Rigurosa:**

| Campo           | Descripción                                                                                          |
|-----------------|------------------------------------------------------------------------------------------------------|
| **Descripción** | Implementaremos una política de seguridad rigurosa que incluye autenticación de dos factores y cifrado de datos en reposo y en tránsito. |
| **Sustento**    | Proteger la información del usuario y cumplir con las regulaciones de seguridad.                     |

**Lenguajes de Programación Específicos:**

| Campo           | Descripción                                                                                          |
|-----------------|------------------------------------------------------------------------------------------------------|
| **Descripción** | El desarrollo del frontend se realizará utilizando ReactJS y el backend con Node.js.                |
| **Sustento**    | Estos lenguajes y frameworks proporcionan un alto rendimiento, escalabilidad y son ampliamente soportados. |

**Estandarización de APIs:**

| Campo           | Descripción                                                                                          |
|-----------------|------------------------------------------------------------------------------------------------------|
| **Descripción** | Todas las interacciones con servicios externos deben realizarse a través de APIs RESTful.            |
| **Sustento**    | Las APIs RESTful son estándar, fáciles de mantener y permiten una integración eficiente con otros sistemas. |
